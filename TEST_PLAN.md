## Test Plan

### 1. 실행 스모크 테스트

- [x] Godot 4.5.1에서 프로젝트가 실행된다.
- [x] `scripts/main.gd`, `scripts/stage_view.gd`가 경고/오류 없이 로드된다.
- [ ] macOS GUI에서 위젯 창이 실제로 보인다.
- [ ] Windows 빌드에서 위젯 창이 실제로 보인다.

### 2. 위젯/관리 화면

- [ ] 기본 실행 시 작은 위젯 화면에는 돌 장면이 중심으로 표시된다.
- [ ] `관리` 버튼을 누르면 창이 `760x720px` 관리 화면으로 커진다.
- [ ] 관리 화면에서 도감/꾸미기/설정 탭을 사용할 수 있다.
- [ ] `돌 보기` 버튼을 누르면 작은 위젯 화면으로 돌아간다.
- [ ] 관리 화면 크기는 위젯 창 크기 저장값을 덮어쓰지 않는다.
- [ ] 항상 위 표시가 기본으로 켜진다.
- [ ] 설정에서 항상 위 표시를 끌 수 있다.

### 3. 성장

- [ ] 게임이 켜져 있는 동안 `active_seconds`가 증가한다.
- [ ] 게임을 껐다 켜면 꺼져 있던 시간은 성장에 반영되지 않는다.
- [ ] 성장 공식 `1.0 + 0.35 * log(1 + active_seconds / 1800)`이 적용된다.
- [ ] 장시간 성장해도 돌이 위젯 구도를 심하게 깨지 않는다.

### 4. 도감 발견

- [ ] 1시간마다 도감 발견 판정이 실행된다.
- [ ] 이미 등록된 돌/꾸미기 항목은 다시 나오지 않는다.
- [ ] 돌 발견 시 현재 메인 돌은 교체되지 않는다.
- [ ] 꾸미기 발견 시 도감에 등록된다.
- [ ] 비어 있는 꾸미기 레이어는 새로 발견한 항목을 자동 장착할 수 있다.
- [ ] 모든 도감 항목을 모으면 추가 지급이 발생하지 않는다.

### 5. 꾸미기

- [ ] 배경/장식장/받침은 각각 1개씩 장착된다.
- [ ] 액세서리 1/2/3 슬롯은 각각 독립적으로 장착된다.
- [ ] 꾸미기 상태가 위젯 화면과 관리 화면에 모두 반영된다.
- [ ] 저장 후 재실행해도 꾸미기 상태가 유지된다.

### 6. 저장/스크린샷

- [ ] `user://save.json`에 로컬 세이브가 생성된다.
- [ ] 창 크기, 항상 위 설정, 도감, 꾸미기 배치가 저장된다.
- [ ] 스크린샷 저장 버튼이 PNG 파일을 생성한다.
- [ ] 저장 파일이 손상되거나 없을 때 기본 상태로 시작한다.
