# 돌 키우기 PRD 초안

상태: Draft  
대상 플랫폼: Windows, macOS  
출시 채널: Steam, Epic Games Store  
배포 정책: GitHub 오픈소스 공개 + Steam/Epic 유료 빌드 판매  
문서 목적: 확정 기획서가 아니라, 개발/검증/출시 논의를 시작하기 위한 첫 기준선

## 1. 제품 개요

`돌 키우기`는 플레이어가 다른 작업을 하는 동안 화면 한쪽에 작게 띄워두는 데스크톱 위젯형 PC 방치 게임이다. 핵심 경험은 "옆에 놓아둔 작은 돌이 아무것도 안 해도 조금씩 커지는 것", "1시간마다 새 돌 또는 꾸미기 아이템을 발견해 도감을 채우는 것", "돌 받침, 배경, 액세서리로 내 돌을 캐주얼하게 꾸미는 것"이다. 플레이어가 키우던 돌은 새 돌 획득으로 교체되지 않고 계속 유지되며, 새 돌과 꾸미기 아이템은 도감에 추가된다.

## 2. 목표

### 제품 목표

- 다른 작업을 방해하지 않는 작은 위젯형 방치 경험과 도감 수집의 장기 목표를 결합한다.
- Windows와 macOS에서 같은 세이브, 같은 밸런스, 같은 콘텐츠 경험을 제공한다.
- Steam과 Epic Games Store 모두에 출시 가능한 PC 패키지 구조를 전제로 개발한다.
- GitHub에는 오픈소스로 공개하고, 직접 빌드하기 싫은 사용자는 Steam/Epic에서 저렴하게 구매할 수 있게 한다.
- 짧은 확인만으로도 성장, 새 도감 진행, 꾸미기 변화를 느끼게 하고, 장기적으로는 돌 도감과 꾸미기 도감 완성을 목표로 한다.
- 캐주얼한 시각 스타일과 꾸미기 요소로 스크린샷을 찍고 공유하고 싶은 장면을 만든다.
- 그래픽은 캐주얼 2D 일러스트 형식을 기본으로 한다.
- 돌 아트는 손그림 느낌의 굵은 외곽선, 약간 과장된 색, 3/4 시점의 독립 오브젝트 스타일을 기준으로 한다.

### 성공 지표 초안

- D1 리텐션: 35% 이상
- D7 리텐션: 15% 이상
- 평균 첫 세션: 8분 이상
- 첫 30분 내 핵심 루프 이해율: 80% 이상
- 위시리스트 전환율: Steam Coming Soon 페이지 방문 대비 10% 이상
- 출시 후 치명적 크래시율: 세션 기준 1% 미만

## 3. 타겟 사용자

### 1차 사용자

- 게임을 켜놓고 대충 다른 작업을 하는 사용자
- 작업/공부/영상 시청 중 화면 구석에 작은 게임을 띄워두고 싶은 사용자
- 저부담 방치형 수집 게임을 선호하는 PC 플레이어

### 2차 사용자

- Steam에서 저가 인디 게임, 힐링 게임, 실험적 시뮬레이션을 탐색하는 사용자
- 캐주얼 꾸미기, 미니어처, 데스크테리어 감성을 좋아하는 사용자
- 마음에 드는 위젯 장면을 스크린샷으로 공유하고 싶은 사용자

## 4. 핵심 게임 루프

1. 플레이어가 돌 상태를 확인한다.
2. 작은 위젯 창을 화면 옆에 둔 채 다른 작업을 한다.
3. 켜둔 시간에 따라 현재 돌이 자동으로 성장한다.
4. 1시간마다 아직 도감에 없는 새 돌 또는 꾸미기 아이템이 우선적으로 발견된다.
5. 발견된 새 돌은 현재 키우는 돌을 대체하지 않고 도감에 추가된다.
6. 평소 위젯 화면에는 돌 장면만 크게 표시한다.
7. 플레이어가 관리 화면을 열면 창이 커지고 도감/꾸미기/설정 UI가 표시된다.
8. 플레이어는 관리 화면에서 돌 받침, 배경, 액세서리로 위젯 장면을 꾸민다.
9. 마음에 드는 상태를 스크린샷으로 저장하거나 SNS 공유용 이미지로 내보낸다.

## 7. 주요 시스템 초안

### 성장 시스템

돌은 플레이어가 게임을 켜둔 시간에 따라 자동으로 성장한다. 게임이 꺼져 있는 동안에는 성장하지 않는다. 별도의 기본 관리 행동은 첫 출시 범위에 포함하지 않는다. 성장은 초반에 빠르게 체감되고 시간이 지날수록 둔화되지만, 완전히 멈추지는 않는 로그형 무한 성장으로 설계한다.

성장 공식 초안:

- `size_multiplier = 1.0 + 0.35 * log(1 + active_seconds / 1800)`
- `active_seconds`는 게임이 켜져 있는 동안 누적되는 시간이다.
- `log`는 자연로그를 사용한다.
- `1800`은 30분을 의미한다.
- 시작 크기는 1.0x다.
- 성장값에는 상한을 두지 않는다.
- 초반 성장 체감이 중요하므로 첫 5분, 30분, 2시간 구간의 성장 곡선을 별도 밸런싱한다.
- 성장값은 돌의 크기에만 직접 반영한다.

초기 기준값:

- 0분: 1.00x
- 5분: 약 1.05x
- 30분: 약 1.24x
- 1시간: 약 1.38x
- 2시간: 약 1.56x
- 10시간: 약 2.07x
- 40시간: 약 2.53x

검증 필요:

- 실제 화면에서 30분 이전 성장이 충분히 체감되는지 확인한다.
- 40시간 이후에도 너무 커져서 위젯 구도를 깨지 않는지 확인한다.
- 필요하면 `scale` 또는 `time_unit` 값을 조정한다.

### 도감 발견

- 1시간마다 도감 발견 판정을 수행한다.
- 발견 대상은 도감에 아직 없는 돌 또는 꾸미기 아이템을 우선한다.
- 도감에 없는 항목이 남아 있다면 중복 항목은 나오지 않는다.
- 모든 항목을 이미 발견했다면 새 항목을 억지로 지급하지 않는다.
- 새 돌 발견은 현재 키우는 돌을 바꾸지 않는다.
- 발견된 돌 또는 꾸미기 아이템은 도감에 등록되며, 현재 메인 돌은 계속 성장한다.
- 돌은 단일 카테고리로 분류하지 않고 여러 개의 태그를 가진다.
- 태그는 도감 필터, 발견 풀 구성, 검색, 공유 문구에 사용한다.

발견 후보 선택 규칙 초안:

1. 전체 도감 항목에서 이미 도감에 등록된 항목을 제외한다.
2. 남은 항목이 없으면 발견 이벤트를 발생시키지 않는다.
3. 남은 항목 중 현재 등장 조건을 만족하는 항목만 후보로 둔다.
4. 후보가 없으면 발견 이벤트를 보류한다.
5. 후보 중 하나를 희귀도/가중치에 따라 선택한다.

도감 완성 시간:

- 전체 도감 완성 예상 시간은 `전체 도감 항목 수 * 1시간`이다.
- 전체 도감 항목 수는 돌 도감 항목 수와 꾸미기 도감 항목 수를 합산한다.

### 도감/컬렉션

- 도감은 수집 진행의 핵심 목표이며, 돌 도감과 꾸미기 도감으로 나뉜다.
- 각 돌은 이름, 외형, 발견 시각, 짧은 설명, 태그 목록, 희귀도를 가진다.
- 각 꾸미기 아이템은 이름, 레이어, 외형, 발견 시각, 짧은 설명, 희귀도를 가진다.
- 중복 획득으로 인한 불쾌감을 피하기 위해 첫 출시 버전에서는 중복 보상을 제공하지 않는다.
- 더 이상 발견할 항목이 없을 때는 "새 항목 없음" 상태를 조용하게 처리하고, 현재 돌 성장 관찰에 집중하게 한다.

### 위젯 창

- 기본 플레이 화면은 작은 독립 창이다.
- 플레이어가 다른 작업 중에도 화면 옆에 배치할 수 있어야 한다.
- 기본 위젯 화면은 돌 장면을 중심으로 표시하고, 도감/꾸미기/설정 UI를 상시 노출하지 않는다.
- 기본 위젯 화면에는 관리 화면으로 진입하는 작은 버튼만 둔다.
- 기본 창 크기는 작업 화면을 방해하지 않을 만큼 작아야 한다.
- 기본 창 크기 권장값은 320x420px이다.
- 최소 창 크기 권장값은 240x320px이다.
- 최대 창 크기 권장값은 480x640px이다.
- 플레이어는 설정에서 위젯 창 크기를 조절할 수 있어야 한다.
- 창 위치와 크기는 종료 후에도 유지한다.
- 항상 위 표시는 기본값으로 켠다.
- 플레이어는 설정에서 항상 위 표시를 끌 수 있다.
- 첫 출시 버전에서는 일반 창 테두리를 허용하되, 장기적으로는 프레임리스/투명 배경 모드를 검토한다.
- 위젯 상태에서도 성장, 도감 발견, 꾸미기 상태가 정상 반영되어야 한다.

### 관리 화면

- 도감, 꾸미기, 설정은 기본 위젯 화면이 아니라 관리 화면에서 조작한다.
- 관리 화면은 위젯 창보다 크게 표시해 도감 목록과 꾸미기 선택을 편하게 만든다.
- 관리 화면 권장 크기는 760x720px이다.
- 관리 화면에서 `돌 보기`를 누르면 작은 위젯 화면으로 돌아간다.
- 관리 화면 크기는 위젯 창 크기 저장값을 덮어쓰지 않는다.

### 꾸미기

- 꾸미기는 돌 자체를 바꾸는 것이 아니라 돌을 보여주는 장면을 꾸미는 시스템이다.
- 꾸미기는 고정된 레이어/슬롯 구조를 가진다.
- 레이어 순서는 뒤에서 앞으로 렌더링한다.
  1. 배경 레이어
  2. 돌 장식장 레이어
  3. 돌 받침 레이어
  4. 돌 레이어
  5. 돌 액세서리 1
  6. 돌 액세서리 2
  7. 돌 액세서리 3
- 배경, 장식장, 받침은 각각 1개씩만 장착한다.
- 돌 액세서리는 최대 3개까지 장착한다.
- 첫 출시 기준 꾸미기 도감 항목 수는 파트별 5개다.
  - 배경 5개
  - 돌 장식장 5개
  - 돌 받침 5개
  - 돌 액세서리 1 슬롯용 5개
  - 돌 액세서리 2 슬롯용 5개
  - 돌 액세서리 3 슬롯용 5개
- 액세서리 슬롯은 서로 다른 위치/용도를 가질 수 있다.
  - 예: 머리 위, 얼굴/전면, 옆/후면
- 꾸미기 아이템은 돌과 같은 도감 발견 풀에서 뽑혀 도감에 등록된다.
- 액세서리는 돌의 정체성을 해치지 않는 캐주얼한 범위로 제한한다.
- 꾸미기 상태는 스크린샷과 공유 이미지에 반영된다.

### AI 대화 (출시 후 후보)

- 기본적으로 돌은 대사하지 않는다.
- AI 대화는 필수 기능이 아니라 선택형 기능이다.
- 플레이어가 옵션에서 직접 AI API 키를 입력한 경우에만 AI 대화를 사용할 수 있다.
- 플레이어는 돌의 성격을 설정할 수 있다.
- AI 대화는 성장, 도감 발견, 꾸미기 도감 등록에 영향을 주지 않는다.
- API 키는 로컬에만 저장한다.

### 스크린샷/공유

- 첫 출시 버전에서는 온라인 연동 없이 로컬 스크린샷 저장을 지원한다.
- SNS 직접 업로드는 필수 기능이 아니다.
- 공유용 이미지는 돌 이름, 성장 수치, 도감 완성도, 현재 꾸미기 상태를 포함할 수 있다.
- 공유 동선은 사용자가 직접 파일을 업로드하는 방식으로 시작한다.
- 첫 출시 버전에는 플랫폼 공유 버튼을 넣지 않는다.

### 경제

- 별도의 재화 경제는 앞으로도 두지 않는다.
- 핵심 진행 가치는 게임을 켜둔 시간, 현재 돌의 누적 성장, 도감 완성도에서 나온다.
- 게임 내 재화, 상점, 구매형 성장 재료, 반복 과금 구조는 범위 밖이다.
- 출시 가격은 USD 2.00을 기본값으로 한다.
- 별도 데모는 제공하지 않는다.
- 소스 코드는 GitHub에 공개한다.
- 스토어 빌드는 직접 빌드하기 싫은 사용자를 위한 유료 편의 배포로 포지셔닝한다.

### 세이브

- 첫 출시 버전은 로컬 세이브만 지원한다.
- 자체 서버, 자체 계정, Steam Cloud, Epic Cloud Saves는 첫 출시 범위에 포함하지 않는다.
- 저장 파일은 Windows/macOS 표준 사용자 데이터 경로에 둔다.
- 저장 파일은 사람이 직접 옮길 수 있는 단일 파일 또는 작은 파일 묶음으로 관리한다.
- 자동 저장과 백업 슬롯을 제공해 위젯형 장시간 실행 중 데이터 손실을 줄인다.
- 플랫폼 클라우드 세이브는 출시 후 필요성이 확인될 때 별도 기획으로 검토한다.

## 9. 사용자 플로우

### 첫 실행

1. 언어와 기본 설정을 선택한다.
2. 첫 돌을 받는다.
3. 돌 이름을 정한다.
4. 위젯 창을 화면 옆에 배치한다.
5. 게임을 켜두면 돌이 자동으로 성장한다는 안내를 본다.
6. 관리 화면을 열어 첫 번째 받침/배경 꾸미기를 적용한다.
7. 관리 화면을 닫고 돌만 보이는 위젯 상태로 둔다.
8. 1시간마다 새 돌 또는 꾸미기 아이템이 도감에 등록된다는 안내를 확인한다.

### 일반 접속

1. 현재 돌의 성장 상태를 관찰한다.
2. 누적 성장량과 변화 기록을 확인한다.
3. 필요할 때 관리 화면을 연다.
4. 도감 발견 타이머 또는 도감 진행을 확인한다.
5. 새 도감 항목이 발견되면 도감 등록 연출을 본다.
6. 현재 돌이 교체되지 않고 계속 성장하는 것을 확인한다.
7. 새로 발견된 꾸미기 아이템이 있으면 위젯 장면에 적용한다.
8. 관리 화면을 닫고 다시 화면 옆에 켜둔다.

### 꾸미기/공유

1. 받침, 배경, 액세서리 메뉴를 연다.
2. 도감에 등록된 꾸미기 아이템을 선택해 현재 돌 장면에 적용한다.
3. 마음에 드는 장면을 로컬 스크린샷으로 저장한다.
4. 필요하면 SNS 공유용 이미지 포맷으로 내보낸 뒤 사용자가 직접 업로드한다.

### 장기 목표 달성

1. 현재 돌이 장기간 누적 성장한다.
2. 도감에 없는 돌 또는 꾸미기 아이템이 1시간마다 하나씩 발견된다.
3. 도감 완성도가 올라간다.
4. 발견 가능한 도감 항목을 모두 모으면 새 항목 지급이 멈춘다.
5. 플레이어는 현재 돌의 무한 성장과 기록 갱신을 계속 관찰한다.

## 10. 플랫폼/출시 요구사항 초안

### Windows

- Windows 10 이상 지원을 기본 목표로 한다.
- x64 빌드를 기본으로 한다.
- 키보드/마우스 조작을 1차 입력으로 한다.
- 작은 창, 창 위치 저장, 항상 위 표시 동작을 검증한다.

### macOS

- Apple Silicon과 Intel Mac 지원 범위는 엔진/빌드 검증 후 확정한다.
- notarization, 권한, 저장 경로 정책은 출시 전 별도 체크리스트로 관리한다.
- 작은 창, 창 위치 저장, 항상 위 표시 동작을 검증한다.

### 엔진/기술 스택

- 엔진은 Godot 4.x를 사용한다.
- 1차 구현 언어는 GDScript를 기본으로 한다.
- 창 제어, 저장 경로, 스크린샷 저장, Windows/macOS export는 Godot 표준 기능을 우선 사용한다.
- Godot MCP를 사용해 프로젝트 생성, 씬 구성, 실행 검증을 진행한다.

### Steam

- Steam Coming Soon 페이지와 빌드 리뷰 일정을 개발 마일스톤에 포함한다.
- Steam 공식 문서 기준, 출시 전 스토어 페이지와 게임 빌드 체크리스트를 각각 제출/승인받아야 한다.
- Coming Soon 페이지는 출시 전 최소 2주 이상 공개되어야 하는 요건을 일정에 반영한다.
- 스크린샷, 트레이딩 카드 등 Steam 기능은 출시 후보 기능으로 우선순위를 정한다.
- Steam Cloud는 첫 출시 범위에서 제외한다.
- Steam Next Fest는 참여하지 않는다.
- 이유: Steam Next Fest는 공개 플레이 가능 데모가 필요한 이벤트인데, 이 프로젝트는 별도 데모 없이 오픈소스 공개 + 저가 유료 빌드 판매 전략을 사용한다.

### Epic Games Store

- Epic Games Store 퍼블리싱 계정, 세금/지급 정보, 빌드 업로드 플로우를 별도 운영 체크리스트로 관리한다.
- Epic 공식 퍼블리싱 안내 기준, 제품은 설명과 일치하게 실행되어야 한다.
- 멀티플레이가 없으면 PC 스토어 간 크로스플레이 요건은 첫 출시 범위에 영향이 없다.
- Epic 업적은 첫 출시 범위에서 제외한다.
- Epic Cloud Saves는 첫 출시 범위에서 제외한다.

## 11. 비기능 요구사항

### 성능

- 일반 플레이 중 60 FPS 목표
- 위젯 대기 상태에서는 CPU/GPU 사용량을 낮게 유지한다.
- 저사양 PC에서 30 FPS 이하로 장시간 떨어지지 않도록 관리
- 첫 실행 로딩 5초 이하 목표
- 저장/불러오기 체감 지연 1초 이하 목표

### 안정성

- 자동 저장은 성장 틱, 도감 발견, 설정 변경 후 수행한다.
- 저장 파일 손상을 대비해 최근 백업 슬롯을 유지한다.
- 성장 시간 계산은 로컬 시간 변경 악용을 고려한다.
- 창 위치가 모니터 변경 후 화면 밖으로 나가지 않도록 복구한다.

### 접근성

- 모든 핵심 확인/설정 행동은 마우스만으로 가능해야 한다.
- 색상만으로 상태를 구분하지 않는다.
- 반복 클릭이나 관리 행동이 성장 효율에 필수가 되지 않도록 한다.
- 자막/텍스트는 UI 스케일 변경을 고려한다.

### 현지화

- 첫 출시 텍스트는 한국어와 영어를 기본으로 설계한다.
- 모든 표시 문자열은 코드와 분리한다.
- 돌 이름, 도감 설명, 이벤트 문구는 현지화 키 기반으로 관리한다.

## 14. 리스크

- 방치형 루프가 너무 단조로우면 첫날 이탈이 높다.
- 플레이어 행동이 거의 없기 때문에 돌 크기 변화와 성장 연출이 약하면 지루해질 수 있다.
- 도감 발견 주기 1시간이 너무 길게 느껴지면 도감 진행감이 약해질 수 있다.
- 도감 발견 주기 1시간 기준으로도 도감 항목 수가 적으면 콘텐츠가 빨리 고갈된다.
- 모든 돌을 발견한 이후의 상태가 허무하게 느껴질 수 있다.
- 위젯 창이 작업을 방해하면 핵심 사용성이 무너진다.
- 꾸미기 아이템이 부족하면 캐주얼 수집/공유 동기가 약해진다.
- 선택형 AI 대화는 API 키 입력/보관 UX와 비용 오해를 주의해야 한다.
- 항상 위 표시, 프레임리스, 투명 배경 같은 창 기능은 엔진/OS별 제약이 있을 수 있다.
- Windows/macOS 동시 지원은 빌드, 로컬 저장 경로, 입력, 패키징 검증 비용을 늘린다.
- Steam/Epic 동시 출시는 스토어 페이지/빌드 배포 정책 차이 때문에 운영 복잡도가 생긴다.
