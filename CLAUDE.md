## Harness-first workflow

이 프로젝트는 “메인 에이전트 1명”이 전부 구현하는 방식 대신, **역할 분리된 에이전트 팀**이 협업하도록 설계합니다.

### 원칙

- **역할 분리**: 아키텍트/백엔드/프론트/QA가 각각 책임 범위를 가진다.
- **인터페이스 우선**: API shape(요청/응답 DTO), DB 스키마, 이벤트/큐 계약 등 “경계”를 먼저 고정한다.
- **품질 게이트**: 통합 전에 체크리스트/테스트로 교차 검증한다.
- **컨텍스트 관리**: 큰 작업은 서브에이전트로 분리하고, 메인 컨텍스트는 “결정/요약/계약”만 유지한다.

- 오케스트레이션은 `.claude/skills/project-orchestrator/SKILL.md`
